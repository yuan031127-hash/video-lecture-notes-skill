# 交付前质量检查

## 目录

1. 结构
2. 概念
3. 公式
4. 视频对齐
5. 画面证据与最终输出
6. 可读性
7. 文件完整性
8. 禁止事项

完成讲义后逐项检查。发现问题时先修正，再报告完成。

## 1. 结构

- [ ] H1 标题后的第一个可见内容是对应原视频的可点击链接。
- [ ] 每章包含“本节解决什么问题”。
- [ ] 每章包含必要的前置储备知识。
- [ ] 每章包含核心概念与公式推导。
- [ ] 每章包含物理图像与直观理解。
- [ ] 章节覆盖全部视频范围，没有因时长而漏掉后半部分。
- [ ] 多视频的顺序、术语和符号前后一致。

## 2. 概念

- [ ] 没有未解释就直接使用的核心概念。
- [ ] 核心术语首次出现时附有英文名称。
- [ ] 容易混淆的相近概念已明确区分。
- [ ] 定义、模型假设和适用条件没有混写。
- [ ] 外部补充没有伪装成教师原话。

## 3. 公式

逐个重要公式确认：

- [ ] LaTeX 能够渲染，括号和定界符匹配。
- [ ] 行内公式统一使用 `$...$`，没有使用容易被转义损坏的 `\(...\)`。
- [ ] Markdown 中不存在由 `\beta`、`\frac` 等误转义产生的控制字符。
- [ ] 上下标、向量和标量书写正确。
- [ ] 每个变量、常数和下标都有解释。
- [ ] 单位正确且等式两边量纲一致。
- [ ] 推导没有省略关键代数或物理步骤。
- [ ] 正负号与坐标约定一致。
- [ ] 使用近似时写明理由和影响。
- [ ] 最终结果通过合理极限检查。
- [ ] 适用条件与失效情形完整。
- [ ] 补充推导已明确标记。

## 4. 视频对齐

- [ ] 章节和主要知识点时间戳准确。
- [ ] 时间戳指向真正开始讲解的位置，而非公式短暂出现的位置。
- [ ] 关键帧与当前讲义内容对应。
- [ ] 画面公式与讲义公式一致。
- [ ] 视频口述与 PPT 的冲突已记录。
- [ ] 疑似口误和无法确认内容已标记。

## 5. 画面证据与最终输出

- [ ] 制作阶段已用关键帧或原视频核对公式、板书和重要图像。
- [ ] OCR 结果已经回到原画面复核。
- [ ] 正文用文字解释重要曲线的坐标、符号、特征点和教师结论。
- [ ] 画面相关说明附有对应视频时间戳。
- [ ] 用户未明确要求图片时，最终讲义没有图片嵌入，课程库没有图片文件。

## 6. 可读性

- [ ] 讲义可以脱离视频连续阅读。
- [ ] 正文不是字幕堆砌或口语记录。
- [ ] 推导前说明目标，推导后解释物理意义。
- [ ] 前置知识只补充当前内容确实需要的部分。
- [ ] 连续解释没有被大量项目符号取代。
- [ ] 类比后说明了类比边界。
- [ ] 术语、符号、单位和标题格式一致。

## 7. 文件完整性

- [ ] `lecture-notes.md` 存在且不是空文件。
- [ ] 课程入口、讲义和概念卡片之外，没有默认交付内部字幕、关键帧或分析目录。
- [ ] 最终 Obsidian 目录中不存在未经要求的图片或其他非 Markdown 文件。
- [ ] 重复使用的核心概念已拆为独立卡片，并从讲义正确双链。
- [ ] 所有 Obsidian 双链目标都存在。
- [ ] 所有待核对项可追溯到时间戳或材料位置。
- [ ] 最终回复列出处理范围、配套材料、生成文件和剩余不确定项。

## 8. 禁止事项

出现任意一项都不能交付：

1. 只总结字幕而忽略画面；
2. 直接给出最终公式而不解释来源；
3. 省略关键推导步骤或使用未定义变量；
4. 不写单位、量纲、假设或适用范围；
5. 无标记地修正口误或猜测模糊公式；
6. 编造视频中不存在的结论；
7. 把模型补充伪装成视频内容；
8. 用大量列表代替连续解释；
9. 用纯文本写复杂公式或混用全角半角符号；
10. 因视频过长而只处理部分内容；
11. 把物理图像写成公式的重复翻译；
12. 未经要求把完整课程压缩成应试提纲；
13. 为追求简洁而牺牲推导完整性。
14. 把内部字幕、截图、关键帧或分析目录默认交付到 Obsidian。
15. 把视频链接埋在元数据或后文章节，而不是放在标题正下方。
16. 用户没有要求图片时仍嵌入封面、板书或课程截图。
